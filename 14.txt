# 🔍 What the Plugin Audit Log Tracks

Each plugin action generates a structured audit entry:

- **Plugin name**  
- **Plugin type** (site / similarity / other)  
- **Method invoked** (`compare`, `extract`, `build_urls`, etc.)  
- **Timestamp**  
- **Execution time**  
- **Sandbox result** (success / timeout / crash / permission violation)  
- **Permissions used**  
- **Permissions blocked**  
- **Arguments summary** (safe, non‑sensitive)  
- **Output summary**  
- **Risk level**  
- **Manifest version**  

This gives you a complete forensic trail.

# 🧩 1. Audit Log Data Model

```python
# core/audit_log.py
from dataclasses import dataclass, field
from datetime import datetime

@dataclass
class AuditEntry:
    timestamp: str
    plugin_name: str
    plugin_type: str
    method: str
    status: str
    duration_ms: int
    permissions_used: dict
    permissions_blocked: dict
    manifest_version: str
    args_summary: dict
    result_summary: dict
    error: str | None = None
```

# ⚙️ 2. Audit Log Writer

```python
import json
import time

class AuditLogger:
    def __init__(self, path="audit.log"):
        self.path = path

    def write(self, entry: AuditEntry):
        with open(self.path, "a") as f:
            f.write(json.dumps(entry.__dict__) + "\n")
```

This writes **one JSON line per event**, perfect for:

- Splunk  
- Elastic  
- Loki  
- SIEM tools  
- jq  
- grep

# 🔒 3. Integrating Audit Logging Into the Sandbox

Wrap every sandboxed plugin call:

```python
# core/sandbox.py (extended)
from core.audit_log import AuditEntry, AuditLogger

logger = AuditLogger()

def sandboxed_call(plugin, method_name, *args, **kwargs):
    permissions = plugin.manifest.get("permissions", {})
    start = time.time()

    result = run_in_sandbox(
        getattr(plugin, method_name),
        args,
        kwargs,
        permissions
    )

    duration = int((time.time() - start) * 1000)

    entry = AuditEntry(
        timestamp=datetime.utcnow().isoformat(),
        plugin_name=plugin.name,
        plugin_type=plugin.manifest.get("type", "unknown"),
        method=method_name,
        status="success" if result.get("ok") else "error",
        duration_ms=duration,
        permissions_used={k: v for k, v in permissions.items() if v},
        permissions_blocked={k: v for k, v in permissions.items() if not v},
        manifest_version=plugin.manifest.get("version", "0.0.0"),
        args_summary={"count": len(args)},
        result_summary={"ok": result.get("ok")},
        error=result.get("error")
    )

    logger.write(entry)
    return result
```

Every plugin action is now logged.

# 🧠 4. Audit Log Viewer API

Expose logs to your UI:

```python
# core/api.py
def get_audit_log(limit=200):
    entries = []
    with open("audit.log") as f:
        for line in f.readlines()[-limit:]:
            entries.append(json.loads(line))
    return entries
```

# 🖥️ 5. Audit Log UI (Frontend)

Add a new tab to your Permission Inspector:

```html
<h2>Plugin Audit Log</h2>
<table id="audit-table">
  <thead>
    <tr>
      <th>Time</th>
      <th>Plugin</th>
      <th>Method</th>
      <th>Status</th>
      <th>Duration</th>
      <th>Risk</th>

  </thead>

<script>
async function loadAuditLog() {
  const res = await fetch('/api/audit');
  const logs = await res.json();

  const tbody = document.querySelector('#audit-table tbody');
  tbody.innerHTML = '';

  logs.forEach(entry => {
    const tr = document.createElement('tr');
    tr.innerHTML = `
      <td>${entry.timestamp}</td>
      <td>${entry.plugin_name}</td>
      <td>${entry.method}</td>
      <td>${entry.status}</td>
      <td>${entry.duration_ms} ms</td>
      <td>${entry.permissions_used.network ? 'medium' : 'low'}</td>
    `;
    tbody.appendChild(tr);
  });
}

loadAuditLog();
</script>
```

# 🔥 6. Security Features Built Into the Audit Log

### ✔ Detect misbehaving plugins  
- repeated timeouts  
- repeated crashes  
- repeated permission violations  

### ✔ Detect suspicious behavior  
- plugin suddenly requesting filesystem access  
- plugin suddenly taking 10× longer to run  
- plugin output size spikes  

### ✔ Detect plugin tampering  
- manifest version mismatch  
- plugin code changed without manifest update  

### ✔ Detect sandbox escapes  
- any blocked permission attempt is logged  

### ✔ Detect plugin impersonation  
- plugin name mismatch  
- plugin type mismatch
