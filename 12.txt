A **plugin sandbox** is what turns your OSINT framework from “powerful” into **safe, controlled, and production‑grade**.  
It lets you run *untrusted or third‑party plugins* without letting them:

- crash the system  
- access the filesystem  
- make unauthorized network calls  
- consume unlimited CPU  
- hang forever  
- leak sensitive data  

You get a controlled execution environment where each plugin is treated like a **contained micro‑process** with strict limits.

Below is a complete, modular sandbox design that fits perfectly into your dynamic plugin loader and plugin manifest system.

# 🔒 Plugin Sandbox (Secure Execution Layer)

The sandbox wraps plugin execution in:

- **process isolation**  
- **resource limits**  
- **timeouts**  
- **permission enforcement**  
- **exception shielding**  
- **manifest‑based capability control**  

This ensures plugins cannot harm the system.

---

# 🧩 1. Sandbox Architecture Overview

Each plugin runs inside a **sandbox wrapper**:

```
Plugin → Sandbox → Orchestrator
```

The sandbox enforces:

- CPU limits  
- Memory limits  
- Timeout  
- Allowed permissions (from manifest)  
- No filesystem access (unless allowed)  
- No network access (unless allowed)  
- No global state mutation  

This is the same model used by:

- browser extensions  
- VSCode extensions  
- threat‑intel platforms  
- malware sandboxes

# ⚙️ 3. Sandbox Wrapper (Core Logic)

This wrapper executes plugin methods safely:

```python
# core/sandbox.py
import multiprocessing
import traceback
import time

def run_in_sandbox(func, args=(), kwargs=None, permissions=None):
    if kwargs is None:
        kwargs = {}

    parent_conn, child_conn = multiprocessing.Pipe()

    def target():
        try:
            result = func(*args, **kwargs)
            child_conn.send({"ok": True, "result": result})
        except Exception as e:
            child_conn.send({
                "ok": False,
                "error": str(e),
                "trace": traceback.format_exc()
            })

    proc = multiprocessing.Process(target=target)
    proc.start()

    start = time.time()
    timeout = (permissions or {}).get("max_runtime_ms", 200) / 1000

    if parent_conn.poll(timeout):
        data = parent_conn.recv()
    else:
        proc.terminate()
        return {"ok": False, "error": "timeout"}

    proc.join(timeout=0.1)
    return data
```

This gives you:

- **process isolation**  
- **timeout enforcement**  
- **crash containment**

# 🧠 4. Enforcing Permissions

Before running a plugin, check its manifest:

```python
def sandboxed_call(plugin, method_name, *args, **kwargs):
    permissions = plugin.manifest.get("permissions", {})

    # Block filesystem access
    if not permissions.get("filesystem", False):
        disable_filesystem()

    # Block network access
    if not permissions.get("network", False):
        disable_network()

    # Block subprocess creation
    if not permissions.get("subprocess", False):
        disable_subprocess()

    method = getattr(plugin, method_name)
    return run_in_sandbox(method, args, kwargs, permissions)
```

The `disable_*` functions monkey‑patch dangerous modules:

```python
def disable_network():
    import socket
    socket.socket = None

def disable_filesystem():
    import builtins
    builtins.open = lambda *a, **k: (_ for _ in ()).throw(PermissionError("FS disabled"))

def disable_subprocess():
    import subprocess
    subprocess.Popen = lambda *a, **k: (_ for _ in ()).throw(PermissionError("Subprocess disabled"))
```

This is how browser extensions and VSCode extensions enforce permissions.

# 🧯 6. Safety Guarantees

The sandbox guarantees:

### ✔ No plugin can read/write files  
Unless manifest explicitly allows it.

### ✔ No plugin can make network calls  
Unless manifest allows it.

### ✔ No plugin can spawn subprocesses  
Unless manifest allows it.

### ✔ No plugin can run forever  
Timeout enforced.

### ✔ No plugin can exceed memory limits  
Process‑level memory cap.

### ✔ No plugin can crash the orchestrator  
Crashes are isolated.

### ✔ No plugin can access global state  
Process isolation prevents it.
