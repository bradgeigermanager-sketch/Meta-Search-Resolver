A social‑media resolver is where your OSINT framework really becomes a *full-spectrum identity engine*.  
To make it modular, flexible, and future‑proof, we’ll build it exactly like your GitHub and StackOverflow plugins — but with **two layers of intelligence**:

### **1. Search‑Engine Polling Layer**  
Uses Google/Bing/DDG/Yandex/Brave to find social accounts indirectly.

### **2. Direct‑Browsing Layer**  
Hits the social platform directly (public pages only) and extracts usernames, IDs, bios, and profile URLs.

And we’ll do it all with a **plugin architecture**, so adding new platforms is as simple as dropping in a file.

# 🐦 3. Example Plugin: Twitter/X





```python
# sites/twitter.py
import re
from urllib.parse import quote_plus
from .base import SitePlugin

class TwitterPlugin(SitePlugin):
    name = "twitter"

    def build_urls(self, query: str) -> list[str]:
        q = quote_plus(query)
        return [
            f"https://twitter.com/search?q={q}&src=typed_query",
            f"https://nitter.net/search?f=users&q={q}",  # privacy-friendly mirror
        ]

    def extract(self, html: str) -> list[dict]:
        results = []

        # Pattern: https://twitter.com/<username>
        profile_re = r"https://twitter\.com/([A-Za-z0-9_]{3,20})"
        for username in set(re.findall(profile_re, html)):
            results.append({
                "username": username,
                "profile_url": f"https://twitter.com/{username}",
                "extra": {"type": "profile"}
            })

        return results
```

# 📸 5. Example Plugin: Instagram

import re

from .base import SitePlugin

                "extra": {"type": "profile"}
            })

        return results
