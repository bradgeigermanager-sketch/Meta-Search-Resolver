# 🧩 Modular Similarity Plugin Architecture  
*(Each plugin checks one type of similarity or connection)*

Every similarity plugin follows this interface:

```python
class SimilarityPlugin(ABC):
    name: str

    @abstractmethod
    def compare(self, account_a, account_b) -> dict:
        """
        Returns:
        {
            "score": float (0–1),
            "signals": {...}
        }
        """
```

Your correlation engine loads all similarity plugins and aggregates their scores.

# 🧩 Example Similarity Plugin (Username Similarity)

```python
# similarity/username_similarity.py
from .base import SimilarityPlugin
from rapidfuzz import fuzz

class UsernameSimilarityPlugin(SimilarityPlugin):
    name = "username_similarity"

    def compare(self, a, b):
        score = fuzz.token_sort_ratio(a.username, b.username) / 100
        return {
            "score": score,
            "signals": {
                "username_a": a.username,
                "username_b": b.username
            }
        }
```

Drop it into your similarity plugin registry:

```python
SIMILARITY_PLUGINS = [
    UsernameSimilarityPlugin(),
    # ...more plugins
]
```
