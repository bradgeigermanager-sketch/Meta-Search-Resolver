# 🔌 StackOverflow Plugin (Modular, Drop‑In)

This plugin does three things:

1. Builds URLs for StackOverflow search  
2. Extracts user profile URLs and user IDs  
3. Returns normalized identity objects for scoring  

---

## 🧩 Plugin Code (Python)

```python
# sites/stackoverflow.py
import re
from urllib.parse import quote_plus
from .base import SitePlugin

class StackOverflowPlugin(SitePlugin):
    name = "stackoverflow"

    def build_urls(self, query: str) -> list[str]:
        q = quote_plus(query)
        return [
            f"https://stackoverflow.com/search?q={q}",
            f"https://stackprinter.appspot.com/v/0?service=stackoverflow&question={q}",
        ]

    def extract(self, html: str) -> list[dict]:
        results = []

        # Pattern: https://stackoverflow.com/users/<id>/<username>
        profile_re = r"https://stackoverflow\.com/users/(\d+)/([a-zA-Z0-9\-]+)"

        for user_id, username in set(re.findall(profile_re, html)):
            results.append({
                "username": username,
                "profile_url": f"https://stackoverflow.com/users/{user_id}/{username}",
                "extra": {
                    "user_id": user_id
                }
            })

        return results
```

---

# 🧠 Why this plugin works well

StackOverflow profile URLs follow a strict pattern:

```
https://stackoverflow.com/users/<id>/<username>
```

That makes extraction extremely reliable.

StackOverflow search pages also embed profile links in:

- question lists  
- answer lists  
- comment sections  
- sidebar “top users”  

So even a single search query often yields multiple identity candidates.
