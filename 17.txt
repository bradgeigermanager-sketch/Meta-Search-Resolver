# 🧠 Schema Breakdown (What Each Field Means)

### **Node fields**

| Field | Meaning |
|-------|---------|
| `id` | Unique node ID (`identity:*`, `account:*`, `artifact:*`, `source:*`) |
| `type` | identity / account / artifact / source |
| `label` | Human‑readable label for UI |
| `platform` | For accounts (github, twitter, reddit, etc.) |
| `confidence` | For identity nodes only |
| `attributes` | Arbitrary plugin‑generated metadata |

---

### **Edge fields**

| Field | Meaning |
|-------|---------|
| `id` | Unique edge ID |
| `source` | Node ID |
| `target` | Node ID |
| `type` | Connection type (USERNAME_SIMILARITY, EMAIL_REUSE, etc.) |
| `weight` | Strength of connection (0–1) |
| `plugin` | Which plugin created the edge |
| `evidence` | URLs, snippets, hashes, similarity scores |

---

# 🧩 Supported Node Types

### **Identity nodes**
- represent clusters of accounts  
- have confidence score  
- connect to accounts via `IDENTITY_HAS_ACCOUNT`

### **Account nodes**
- represent platform accounts  
- contain username, profile URL, bio, location, avatar hash, etc.

### **Artifact nodes**
- email  
- domain  
- avatar hash  
- photo hash  
- external link  
- phone fragment  
- EXIF metadata  

### **Source nodes**
- search engine results  
- scraped pages  
- plugin‑specific sources  

---

# 🔗 Supported Edge Types (Plugin‑Generated)

### Identity edges
- `IDENTITY_HAS_ACCOUNT`

### Account‑to‑artifact edges
- `ACCOUNT_USES_EMAIL`
- `ACCOUNT_LINKS_TO_DOMAIN`
- `ACCOUNT_HAS_AVATAR_HASH`
- `ACCOUNT_POSTED_PHOTO_HASH`

### Account‑to‑account edges
- `USERNAME_SIMILARITY`
- `BIO_OVERLAP`
- `LOCATION_MATCH`
- `EXTERNAL_LINK_REUSE`
- `SOCIAL_GRAPH_OVERLAP`
- `CONTENT_REUSE`
- `CO_OCCURRENCE`

### Account‑to‑source edges
- `FOUND_IN_SOURCE`
